# Design Probe

1. Inspect the remaining probe budget and declared menu with `case_status`.
2. Predict each available probe's result under the top competing hypotheses.
3. Choose the lowest-cost probe whose possible outcomes change the ranking or
   candidate repair. Do not probe merely to collect more data.
4. Call `request_probe` only with an exact declared `probe_id`.
5. Bind the immutable probe receipt to the hypothesis update.

Never request physical action, arbitrary host commands, hidden evaluation,
network access, or a probe outside the menu. If no probe discriminates the
alternatives, preserve uncertainty and abstain from a strong causal claim.
